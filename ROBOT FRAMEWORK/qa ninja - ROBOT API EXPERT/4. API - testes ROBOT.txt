INICIANDO AUTOMAÇÃO
--------------------
Uma nova pasta do projeto para executar os testes foi criada.
No VSCODE, dentro do novo projeto vários casos de testes foram criados.

COMANDOS TERMINAL - EXECUTAR TESTES
------------------------------------
	robot post.robot		=	para executar o framwork ROBOT nos casos de testes criados. Dessa forma, os arquivos de retorno serão criados na RAIZ do projeto.
	robot -d .logs post.robot	= 	para executar o framwork ROBOT nos casos de testes criados. Os arquivos de retorno serão salvos em uma pasta separada (logs)
	robot -d ./logs -i bugs tests/post.robot 	= para executar o framwork ROBOT somente nos casos de testes que estão com a [tag] BUGS colocada. Os arquivos de retorno serão salvos em uma pasta separada (logs).

ATENÇÃO
---------
	Para encontrar as bibliotecas do ROBOT acesse (https://robotframework.org/#resources). Para testar API precisamos instalar (HTTP REQUESTLIBRARY (PYTHON)). Ao clicar em cima da biblioteca, o ROBOT nos informa o comando que precisa ser executado no terminal para instalação:     (pip install robotframework-requests)
	Caso fossemos fazer teste na aplicação, a biblioteca ideal seria SELENIUMLIBRARY comando de instalação disponível no site.
	Para utilizar o MONGODB com ROBOT precisamos executar o comando (pip install robot-mongodb-library) e (pip install "pymongo[srv]")


Link biblioteca ROBOT request https://github.com/MarketSquare/robotframework-requests#readme
